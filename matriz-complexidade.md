# 📊 Metodologia de Estimativas - Sprint 1
> Esta matriz de complexidade (1 a 5) aplica-se às Tasks técnicas e de gestão do projeto, garantindo uma métrica de Velocity unificada.

Este documento descreve o critério utilizado pela equipe para mensurar o esforço das atividades planejadas.

## 🟠 Matriz de Complexidade (1 a 5)
Para esta Sprint, optamos por utilizar uma escala de **Complexidade Relativa** representada por níveis de 1 a 5. Essa escolha visa dar agilidade ao processo de refinamento e garantir que todos os membros tenham uma percepção visual clara do desafio técnico de cada funcionalidade.

### 🔍 Descrição dos Níveis:

*   **🟠⚪⚪⚪⚪ (Nível 1):** Tarefas de baixa complexidade. Envolvem ajustes de texto, documentação inicial ou criação de links simples.
*   **🟠🟠⚪⚪⚪ (Nível 2):** Esforço pequeno. Implementações de interface simples e telas estáticas.
*   **🟠🟠🟠⚪⚪ (Nível 3):** Complexidade Média. Exige lógica de programação para filtros básicos e organização de dados.
*   **🟠🟠🟠🟠⚪ (Nível 4):** Complexidade Alta. Envolve múltiplas variáveis, como ordenação de preços ou integração básica de APIs.
*   **🟠🟠🟠🟠🟠 (Nível 5):** Esforço Máximo. Funcionalidades que exigem geolocalização, buscas avançadas (ISBN) ou tratamento de grandes volumes de dados.
